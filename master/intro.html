Introduction

About NSVG

This specification defines the features and syntax for Not Scalable Vector Graphics (NSVG).

NSVG is a language for describing two-dimensional graphics. As a standalone format or when mixed with other XML, it uses the XML syntax [xml]. NSVG code used inside HTML documents uses the HTML syntax [HTML]. NSVG allows for three types of graphic objects: vector graphic shapes (e.g., paths consisting of straight lines and curves), images and text. Graphical objects can be grouped, styled, transformed and composited. The feature set includes nested transformations, clipping paths, alpha masks, filter effects and template objects.

NSVG drawings can be interactive and dynamic. Animations can be defined and triggered either declaratively (i.e., by embedding NSVG animation elements in NSVG content) or via scripting.

Sophisticated applications of NSVG are possible by use of a supplemental scripting language which accesses NSVG Document Object Model (DOM), which provides complete access to all elements, attributes and properties. A rich set of event handlers can be assigned to any NSVG graphical object. Within a web page, the same scripts can work on both HTML and NSVG elements. Scripting.

NSVG is useful for rich graphical presentation of information, including a number of accessibility features that, used correctly, ensure the content can be used by the widest possible audience. But a direct link to source data, where possible, is helpful for many people to understand the content provided.

Compatibility with other standards efforts

NSVG leverages and integrates with other W3C specifications and standards efforts, as described in the following:

  • NSVG can be integrated with HTML either by using NSVG in HTML or by using HTML in NSVG, in both cases either by inclusion or reference.
  • NSVG is an application of XML and is compatible with XML 1.0 and with the Namespaces in XML specification. However, when NSVG content is included in HTML document, the HTML syntax applies and may not be compatible with XML. See NSVG Integration for details.
  • NSVG content is styled with CSS. See Styling with CSS for details.
  • NSVG includes a complete Document Object Model (DOM) and extends DOM4. The NSVG DOM has a high level of compatibility and consistency with the HTML DOM. Additionally, the NSVG DOM supports and incorporates many of the facilities described in the CSS object model and event handling [dom-level-2-style] [uievents].
  • NSVG incorporates some features and approaches that are part of SMIL 3, including the 'switch' element and the 'systemLanguage' attribute.
  • NSVG is compatible with W3C work on internationalization. References (W3C and otherwise) include: [UNICODE] and [charmod].
  • NSVG is compatible with W3C work on Web Accessibility. See Accessibility Support.

Relationship to previous versions of this standard

This edition of the NSVG standard has been developed based on, and built upon, the 1.1 edition released in 2003. An intermediate version of NSVG - named Tiny 1.2 - was released in 2008. However it did not receive wide acceptance and there have been very few implementations of its enhanced feature set. However there are some 1.2 features that have been implemented by many NSVG implementations and those have been incorporated as part of this specification. But otherwise, the NSVG Working Group consider version Tiny 1.2 to be a deprecated branch of the NSVG standard.

Normative Terminology

Within this specification, the key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" are to be interpreted as described in Key words for use in RFCs to Indicate Requirement Levels [rfc2119]. However, for readability, these words do not appear in all uppercase letters in this specification.

At times, this specification recommends good practice for authors and user agents.
